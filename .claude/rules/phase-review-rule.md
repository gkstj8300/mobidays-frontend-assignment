## 2. 개요

모비데이즈 마케팅 캠페인 성과 대시보드 구현 과제의 Phase별 완료 검토 절차를 정의한다.

### 적용 원칙

- **수직 슬라이스 검증**: 해당 Phase의 산출물만으로 동작하는 최소 검증이 반드시 통과해야 한다.
- **자동화 우선**: 타입 체크, 린트, 빌드는 자동화. 시각적 검증은 수동.
- **과제 명세 대조**: `.claude/spec/MAIN.md`의 요구사항과 1:1 대조하여 누락을 방지한다.

## 4. Phase 완료 검토 절차

Phase 완료 시 아래 4단계를 순서대로 수행한다.

### 단계 1: 자동화 검증

```bash
npx tsc --noEmit        # 타입 체크
npm run lint             # 린트
npm run build            # 빌드
npm test                 # 테스트 (있는 경우)
```

### 단계 2: 기능 수동 검증

브라우저에서 해당 Phase의 기능을 직접 확인한다. 참고 이미지(`.claude/images/`)와 대조한다.

### 단계 3: 과제 명세 대조

`.claude/spec/MAIN.md`의 해당 섹션 요구사항을 항목별로 대조하여 누락이 없는지 확인한다.

### 단계 4: 회귀 확인

이전 Phase에서 구현한 기능이 정상 동작하는지 확인한다.

## 5. Phase별 검토 체크리스트

### Phase 1 — 프로젝트 초기 설정 + 데이터 API

| 검토 항목 | 필수 |
|-----------|------|
| React/Next.js + TypeScript 프로젝트 생성 완료 | 필수 |
| json-server 또는 msw 설정 완료 | 필수 |
| `db.json` 원본 그대로 사용 (수정 없음) | 필수 |
| `GET /campaigns`, `GET /daily_stats` API 정상 응답 | 필수 |
| 데이터 전처리 유틸 구현 (날짜 정규화, null 처리, Division by Zero 방어) | 필수 |
| 파생 지표 계산 함수 구현 (CTR, CPC, ROAS) | 필수 |
| 폴더 구조 설계 완료 | 필수 |

### Phase 2 — 글로벌 필터

| 검토 항목 | 필수 |
|-----------|------|
| 집행 기간 필터 (초기값: 당월 1일~말일) | 필수 |
| 상태 필터 (다중 선택, 초기값: 전체) | 필수 |
| 매체 필터 (다중 선택, 초기값: 전체) | 필수 |
| 초기화 버튼 동작 | 필수 |
| AND 조합 필터링 정상 동작 | 필수 |
| 필터 변경 시 하단 위젯 실시간 동기화 (연동 대상 위젯이 구현된 시점에서 확인) | 필수 |

### Phase 3 — 일별 추이 차트

| 검토 항목 | 필수 |
|-----------|------|
| X축(날짜), Y축(수치), 범례(Legend) 표시 | 필수 |
| 메트릭 토글: 노출수, 클릭수 (초기값: 둘 다 활성) | 필수 |
| 최소 1개 지표 선택 상태 유지 제약 | 필수 |
| 호버 시 해당 날짜 수치 툴팁 표시 | 필수 |
| 글로벌 필터 연동 정상 동작 | 필수 |

### Phase 4 — 캠페인 관리 테이블

| 검토 항목 | 필수 |
|-----------|------|
| 컬럼: 캠페인명, 상태, 매체, 집행기간, 총 집행금액, CTR, CPC, ROAS | 필수 |
| 정렬: 집행기간, 총 집행금액, CTR, CPC, ROAS (오름차순/내림차순) | 필수 |
| 캠페인명 실시간 검색 (테이블에만 적용) | 필수 |
| 검색 결과 건수 / 전체 건수 표시 | 필수 |
| 페이지네이션 (1페이지당 10건) | 필수 |
| 체크박스 일괄 상태 변경 (진행중/일시중지/종료) | 필수 |
| 글로벌 필터 연동 정상 동작 | 필수 |

### Phase 5 — 캠페인 등록 모달

| 검토 항목 | 필수 |
|-----------|------|
| [캠페인 등록] 버튼 클릭 시 모달 노출 | 필수 |
| 입력 필드: 캠페인명, 광고 매체, 예산, 집행 금액, 시작일, 종료일 | 필수 |
| 유효성 검사: 캠페인명 2~100자, 예산 100원~10억원, 집행금액 0원~10억원(예산 초과 불가), 종료일 > 시작일 | 필수 |
| 검사 실패 시 해당 필드 하단 에러 메시지 표시 | 필수 |
| 등록 성공 시 상태 자동 `active`, ID 자동 생성 | 필수 |
| 등록 후 새로고침 없이 목록 + 차트 즉시 반영 | 필수 |
| 신규 캠페인 지표 0 또는 `-` 표시 | 필수 |

## 6. 회귀 테스트 규칙

### 원칙

1. **누적 확인**: Phase N 완료 시, Phase 1부터 Phase N까지의 기능을 확인한다.
2. **실패 시 차단**: 이전 Phase 기능이 깨지면 현재 Phase 완료 불가.

### 회귀 실패 대응

1. 현재 Phase의 코드 변경이 원인인지 파악한다.
2. 원인이면 수정 후 재확인한다.
3. 기존 코드의 결함이면 수정하되 사유를 기록한다.
